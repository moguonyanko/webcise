b, i, hr, s, small, u

これらの要素はHTML5で意味合いが変更され再定義された。

  • iは区別される要素。大抵italicで記述されます。
  • bは重要性は小さいが文体は変えたい要素。大抵太字になります。
  • sは正確でなくなった要素。大抵打ち消し線が表示されます。
  • uは伝わりにくい要素。下線が引かれます。
  • hrは意味的な気切りを示します。
smallはコピーライトなど注釈的な要素に対して使います。

table

table関連要素の各種属性

HTML5から多くの属性が非サポートになっている。 特にスタイルを指定する属性が非サポートになっている。 theadはどこに記述しても表のヘッダーとして表示されるが tbodyやtfootよりは先に記述していないとエディタに警告される。 tfootはどこに記述しても表のフッターとして表示され警告もされない。

colgroup要素にはtable要素内共通の設定を記述する。 ページ上に表示されることはない。 プレーンテキストも含めcol以外を記述することはできない。 colgroupもcolもスタイルを指定するためだけの要素にしか思えない。 テーブルの列単位でスタイルを指定するための プロパティがCSSに定義されているべきではないか。 行単位のスタイルはtrに指定すればいい。 colgroupはcaptionより後theadやtbody，tfoot，trより前に 記述されていなければならない。

col要素のstyle属性の値はCSSに記述しても有効。 最初のcolのspanに2と書くと「最初2つの列の設定を指定する」という意味になる。 colのspanとcolの数で算出される列数がtableの列数と一致しないと エディタは警告は出す。以下の例だとtableの列数は4でなければならない。

picture

media属性による表示画像の切り替え

メディアクエリの条件に合っていればsrcsetの画像がダウンロードされる。 このとき後続のimg要素の画像はダウンロードされない。 逆にメディアクエリの条件に合わない場合，srtsetの画像はダウンロードされず img要素の画像がダウンロードされる。 ただしブラウザサイズを変更するなどしてメディアクエリの条件を満たすようになる， あるいは満たさないようになる場合，もう一方の画像ダウンロードが開始され表示される。

picture要素内にimg要素が記述されていないと画像のダウンロードが一切行われない。

解像度で切り替えることもできる。

sizes属性による表示画像の切り替え

sizesはpictureにもimgにも使用できる。

メディアクエリに指定した幅の条件に合致するデバイスでは横長の画像を表示するといったことが可能。
